# 文档说明

## 文档定位

本文档用于补充 `README.md`，帮助首次接触 `EMP-Script-Web` 的开发者快速理解仓库定位、目录职责、核心工作方式以及阅读顺序。

如果需要快速了解项目是什么、如何引入依赖、有哪些主要能力，优先阅读 `README.md`。

如果需要进一步理解这个仓库为什么这样组织、代码应该从哪里开始看、哪些行为由外部配置决定，则阅读本文档。

## 项目定位

`EMP-Script-Web` 是面向 EMP / EasyWeb 体系的企业级 Web **共享业务模块库**。它以 Maven `jar` 的形式提供能力，供宿主 Web 项目引用，而不是一个可独立运行的完整站点工程。

仓库中的 Java 代码主要负责以下几类能力：

- 用户认证、自动登录、短信验证、JWT 鉴权
- 权限控制与多角色访问控制
- HTTP 扩展接口
- 文档、二维码、短网址、上传资源等通用能力
- 微信、小程序、消息通知、任务调度等企业场景能力

需要特别注意的是：很多功能并不是只靠 Java 代码完成的。项目大量依赖 EMP XML 配置、数据库配置以及宿主工程中的 JSP/XML 资源共同驱动运行时行为。

## 仓库结构说明

### 关键目录

#### `src/main/java/com/gdxsoft/web/dao`

数据库实体与 DAO 层。这里的类大多采用统一生成风格：

- `Xxx.java` 表示表映射实体
- `XxxDao.java` 表示对应表的 CRUD 封装

这是理解整个仓库数据访问方式的最佳入口。

#### `src/main/java/com/gdxsoft/web/user`

用户登录、注册、短信验证码、自动登录相关逻辑。

建议重点阅读：

- `RegisterOrLogin.java`
- `DoLogin.java`
- `ValidBase.java`

#### `src/main/java/com/gdxsoft/web/acl`

权限控制与登录状态判断逻辑。

建议重点阅读：

- `Login.java`
- `AclBase.java`
- `AdminImpl.java`
- `BusinessImpl.java`
- `CustomerImpl.java`

#### `src/main/java/com/gdxsoft/api`

对外 API 鉴权能力，尤其是 JWT 的创建和校验。

建议重点阅读：

- `Auth.java`
- `JwtUtils.java`

#### `src/main/java/com/gdxsoft/web/http`

轻量 HTTP 扩展入口。这里的类通常都比较薄，负责：

1. 从 `HttpServletRequest` 构造 `RequestValue`
2. 调用业务类
3. 返回 JSON 或字符串响应

如果要排查一个接口的处理链路，这里通常是入口之一。

#### `src/main/java/com/gdxsoft/web/app`

应用级上下文辅助逻辑，包括：

- 微信 / 小程序 / 原生 App / iPhone / Android 判断
- 语言处理
- 应用配置输出
- 页面运行时环境变量拼装

#### 其他业务目录

- `web/doc`：文档模板、文档生成与文档相关处理
- `web/message`：邮件、短信等消息能力
- `web/job` / `web/schedule`：任务与调度相关逻辑
- `web/shortUrl`：短网址生成与解析
- `web/uploadResources`：上传资源读取和访问控制
- `web/weixin`：微信、小程序相关能力
- `chatRoom`：聊天室 / WebSocket 相关逻辑
- `project`：项目业务相关逻辑
- `spider`：抓取与采集相关逻辑

## 项目的核心工作方式

### 1. 配置驱动

这是本仓库最重要的特点之一。

很多用户界面、登录动作、模块渲染并不是直接在 Java 中硬编码完成的，而是通过：

- `HtmlControl`
- `xmlname`
- `itemname`

这些参数去触发 EMP XML 中定义的页面或动作。

因此在修改登录、注册、模块渲染、文档生成等功能时，不能只看 Java 文件，还要意识到真正的执行逻辑可能来自外部 XML 配置。

### 2. `RequestValue` 作为统一上下文对象

仓库中大量代码使用 `RequestValue` 传递：

- 请求参数
- session / cookie 值
- SQL 参数
- 系统上下文值

也就是说，`RequestValue` 在这里不只是“请求对象包装”，更像一个贯穿页面、业务、数据库和权限逻辑的统一上下文载体。

### 3. DAO 风格统一

DAO 层几乎都遵循相同模式：

- 静态 SQL 字符串
- `FIELD_LIST`
- `KEY_LIST`
- `getRecord(...)`
- `getRecords(...)`
- `createRequestValue(...)`

因此阅读一个 DAO 后，通常就能快速理解其他 DAO 的结构。

### 4. HTTP 层保持轻量

`com.gdxsoft.web.http` 下的类一般不承载复杂业务，而是负责衔接 Servlet 与内部业务逻辑。

如果要新增或调整接口，通常应把核心逻辑放在业务类中，而不是把 HTTP 入口写成庞大的控制器。

## 阅读建议

如果你是第一次阅读这个仓库，建议采用下面的顺序：

1. 先看 `pom.xml`，确认项目是 Java 8 + Maven 的库工程。
2. 再看 `README.md`，了解项目能力边界和使用方式。
3. 然后从 `web/dao` 里挑一组实体与 DAO 理解数据访问模式。
4. 接着阅读 `web/user`、`web/acl`、`api`，理解认证和权限主流程。
5. 再阅读 `web/http`、`web/app`，把握接口入口与应用上下文。
6. 最后按业务方向继续深入 `doc`、`weixin`、`message`、`schedule`、`shortUrl` 等目录。

## 按目标阅读

如果你不是“通读代码库”，而是带着具体问题来接手，可以按下面的目标切入：

### 1. 想搞清楚登录为什么成功 / 失败

优先阅读：

- `user说明.md`
- `acl说明.md`
- `src/main/java/com/gdxsoft/web/user/RegisterOrLogin.java`
- `src/main/java/com/gdxsoft/web/user/DoLogin.java`
- `src/main/java/com/gdxsoft/web/acl/Login.java`

同时要意识到：

- 图形验证码、表单校验、部分登录页面逻辑可能在 EWA XML / `HtmlControl` 对应配置里
- 最终登录态是否被认可，还要看 `AclBase` 是否把值识别为 `SESSION` 或 `COOKIE_ENCYRPT`

### 2. 想排查某个 HTTP 接口为什么返回异常

优先阅读：

- `http说明.md`
- `src/main/java/com/gdxsoft/web/http`
- 被 HTTP 入口调用的业务类

通常排查顺序是：

1. 找到 `IHttp` 实现类
2. 看它从 `RequestValue` 读了哪些参数
3. 找它调到的业务类
4. 再看业务类背后的 DAO / SQL / 外部 XML 配置

### 3. 想改用户、项目、微信等业务数据

优先阅读：

- `dao说明.md`
- 对应模块说明，如 `project说明.md`、`weixin说明.md`

因为这类功能大多是：

- 业务类直接写 SQL
- 或者组合使用生成式 DAO

先理解 DAO 风格，再看业务代码会快很多。

### 4. 想定位小程序 / 微信 / App 页面初始化问题

优先阅读：

- `app说明.md`
- `weixin说明.md`
- `src/main/java/com/gdxsoft/web/app/App.java`
- `src/main/java/com/gdxsoft/web/app/AppMini.java`

这类问题通常和：

- `User-Agent`
- `INIT_CFG`
- `open_id`
- `wx_cfg_no`
- `BAS_WX_CFG`

这些上下文有关。

### 5. 想排查文件、文档、二维码、短链接输出问题

优先阅读：

- `fileviewer说明.md`
- `doc说明.md`
- `shorturl-qrcode说明.md`
- `uploadResources说明.md`

因为这几块都涉及：

- 文件系统路径
- 上传目录
- 图片 / PDF / 二维码缓存

## 模块关系（文本版）

可以用下面这条主线理解仓库中的模块关系：

### 运行时入口层

- `web/http`
- EWA 页面 / `HtmlControl`
- `easyweb.websocket` 接入点

### 通用上下文层

- `RequestValue`
- session / cookie
- EMP XML 配置

### 认证与访问控制层

- `web/user`
- `web/acl`
- `api`

### 基础数据访问层

- `web/dao`
- 少量业务类中的直接 SQL

### 业务能力层

- `project`
- `weixin`
- `message`
- `doc`
- `schedule`
- `shortUrl`
- `qrcode`
- `uploadResources`
- `fileViewer`
- `app`
- `chatRoom`

也就是说，典型链路通常不是“控制器 -> service -> repository”这类现代分层写法，而更接近：

**入口 / XML 配置 -> `RequestValue` -> 业务类 -> DAO / SQL -> 文件系统 / 外部配置 / 第三方服务**

## 常见排查路线

### 1. 页面能打开，但行为不对

优先检查：

- 关联的 Java 业务类
- 对应 `HtmlControl.init(xmlname, itemname, ...)`
- 外部 XML 配置是否和当前参数一致

### 2. 登录态看起来有值，但 ACL 仍然拦截

优先检查：

- `RequestValue` 中该值的 `PageValueTag`
- 是否来自 session 或加密 cookie
- `AclBase.getString(...)` 是否会认可这个来源

### 3. 调度没执行或没有结果

优先检查：

- `schedule说明.md`
- `schedule.xml` 是否存在于部署环境
- `schedule_main`
- `oa_work.schedule_sub`

### 4. 微信、小程序功能异常

优先检查：

- `BAS_WX_CFG`
- 当前 `wx_cfg_no`
- 当前商户上下文
- 是否命中了缓存实例

### 5. 文件 / 二维码 / 上传资源访问不到

优先检查：

- 上传目录物理路径
- 缓存目录是否可写
- 路径是否被安全规则拦截
- 资源是否走到了正确的 HTTP 入口类

## 本地验证说明

建议优先使用以下命令做本地验证：

```bash
mvn -q -DskipTests compile
```

可单独运行的测试包括：

```bash
mvn -q -Dtest=TestResourceConfig test
mvn -q -Dtest=TestWs test
```

完整测试：

```bash
mvn -q test
```

需要注意，完整测试依赖本地 `ewa_conf.xml` / `ewa_conf_console.xml` 以及数据库配置，默认环境下不一定能直接通过。

## 总结

可以把这个仓库理解为：

**EMP / EasyWeb 体系下的企业级 Web 业务能力库**

它的特点不是“前后端分离的完整站点工程”，而是“Java 业务模块 + 配置驱动运行机制 + 宿主工程集成”的组合模式。理解这一点后，再阅读目录结构和业务代码会更容易把握整体脉络。
